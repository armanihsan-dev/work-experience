All About Branches in Git

Definition

A branch in Git is like a separate workspace where you can make changes without affecting the main project. It helps you work on new features or fixes independently. The default branch is usually called main or master.

Commands

1. Create a New Branch

git branch branch_name

2. List All Branches

git branch

Use git branch -a to list all branches, including remote branches.

3. Switch to a Branch

git checkout branch_name

Alternatively, in modern Git versions, use:

git switch branch_name

4. Create and Switch to a New Branch

git checkout -b branch_name

Or, use:

git switch -c branch_name

5. Merge a Branch

git merge branch_name

This command merges the specified branch into the current branch.

6. Delete a Branch

git branch -d branch_name

Use git branch -D branch_name to forcefully delete a branch.

7. Push a Branch to Remote

git push origin branch_name

8. Delete a Remote Branch

git push origin --delete branch_name

Example

Here is an example workflow to demonstrate the use of branches in Git:

  1. Create and switch to a new branch:
    2. git checkout -b feature-login
  2. Make changes and commit them:
    3. git add .
git commit -m "Add login feature"
  3. Switch back to the main branch:
    4. git checkout main
  4. Merge the feature-login branch into main: 
    5. git merge feature-login
  5. Delete the feature-login branch:
    6. git branch -d feature-login

Tips

  • Keep your branches small and focused to make them easier to manage.
  • Always pull the latest changes before creating a new branch to avoid merge conflicts later.
  • Use descriptive names for branches, such as feature-xyz or bugfix-abc.
